# 《基于 OpenID Connect 的 Web 单点登录与访问控制系统复现及 PKCE 调优》项目分工

## A同学——OIDC认证流程与单点登录模块

### 主要职责

负责系统认证模块的设计与实现，完成 OpenID Connect 授权码模式登录流程复现。

### 具体任务

1. 搭建 Authorization Server（认证授权服务器）
2. 搭建 Client（客户端应用）
3. 实现用户登录认证功能
4. 实现 Authorization Code 签发机制
5. 实现 Access Token 与 ID Token 签发
6. 实现 OIDC 授权码模式完整登录流程
7. 完成单点登录（SSO）流程验证

### 论文负责内容

* 第3章 系统设计（OIDC认证架构部分）
* 第4章 系统实现（认证流程实现部分）

### 核心交付物

* OIDC登录流程代码
* 授权码模式时序图
* 系统认证模块设计说明
* 登录演示截图与测试结果

---

## B同学——访问控制与资源授权模块

### 主要职责

负责系统访问控制模型设计与实现，构建基于 JWT、Role 和 Scope 的资源授权机制。

### 具体任务

1. 设计用户角色模型（User/Admin）
2. 设计资源权限模型
3. 设计 Scope 权限体系
4. 实现 JWT 校验功能
5. 实现 Resource Server
6. 实现 Role-Based Access Control（RBAC）
7. 实现 Scope-Based Access Control
8. 实现受保护资源接口（Profile、Email、Admin等）

### 论文负责内容

* 第3章 系统设计（访问控制架构部分）
* 第4章 系统实现（访问控制实现部分）

### 核心交付物

* RBAC权限模型
* Scope授权模型
* JWT校验模块
* 权限测试结果与权限矩阵表

---

## C同学——攻击实验、PKCE增强与性能分析模块

### 主要职责

负责安全分析与实验验证，设计攻击场景并实现 PKCE 安全增强机制。

### 具体任务

1. 实现授权码截获攻击模拟
2. 实现登录CSRF攻击模拟
3. 实现权限提升与Scope滥用测试
4. 实现PKCE机制（Code Verifier/Code Challenge）
5. 实现State参数校验机制
6. 设计安全防护方案
7. 完成攻击前后效果对比实验
8. 完成PKCE性能开销测试与调优分析

### 论文负责内容

* 第5章 攻击与防御实验
* 第6章 PKCE调优实验

### 核心交付物

* Attack Simulator模块
* PKCE实现模块
* 安全实验结果
* 攻击成功率统计表
* 性能分析图表
